# Roadmap: Clickresto Platform

## Overview

Build a dual-portal SaaS platform where restaurant owners discover and request demos while affiliates refer restaurants and track their commissions. Start with secure foundation and lead capture to validate demand, layer in affiliate program for distribution, then enhance CRM and commission workflows for operational excellence.

## Phase Details

### Phase 1: Foundation & Lead Capture
**Goal**: Restaurant owners can request demos, admins can view and manage leads securely
**Depends on**: Nothing (first phase)
**Requirements**: AUTH-01, AUTH-02, AUTH-03, AUTH-04, LAND-01, LAND-02, LAND-03, LAND-04, ADM-01, CRM-02
**Success Criteria** (what must be TRUE):
  1. User can sign up and log in with email/password or magic link
  2. User role determines access (admin sees admin back-office, affiliate sees affiliate portal)
  3. Restaurant owner can submit demo request from landing page
  4. Admin can view all submitted leads in dashboard
  5. Lead data is automatically captured in CRM when demo form submitted

### Phase 2: Affiliate Program
**Goal**: Affiliates can refer restaurants and track their commissions in real-time
**Depends on**: Phase 1
**Requirements**: AFF-01, AFF-02, AFF-03, ADM-02
**Success Criteria** (what must be TRUE):
  1. Affiliate can generate unique referral link from dashboard
  2. Affiliate can view detailed commission history and earnings breakdown
  3. Affiliate can submit payment request for earned commissions
  4. Admin can view all affiliates with stats and approve new affiliates
  5. Leads from affiliate referral links are properly attributed and tracked

### Phase 3: CRM & Commission Workflow
**Goal**: Admins have complete control over lead management, commission validation, and system configuration
**Depends on**: Phase 2
**Requirements**: CRM-01, CRM-03, CRM-04, ADM-03, ADM-04
**Success Criteria** (what must be TRUE):
  1. Admin can view leads in visual pipeline/kanban organized by status
  2. Admin can add notes and view activity timeline for any lead
  3. Admin can search and filter leads by various criteria
  4. Admin can approve or reject payment requests with audit trail
  5. Admin can configure commission amounts and system texts without code changes
